React 데이터 관련 개념

Props (중요⭐️해요!)

Props는 부모 컴포넌트가 자식 컴포넌트에게 값을 전달할때 사용해요!

value=Value 로 값을 넣어주면 되요!

부모 컴포넌트

                    render() {
                        return  // haneum이 출력되죠!
                    }

자식 컴포넌트

                    class MyName extends Component {
                            render() {
                                return (
                                    return <MyName name="리엑트" />
                                );
                        }
                    }

기본 값을 정해주는 방법

자식 컴포넌트 내에서 사용해요!

static defaultProps = { 기본 값 설정해요 ... } 나 자식컴포넌트이름.defaultProps = { 기본 값 설정해요 ... } 로 클래스 내부에 선언해주면 되요!

함수형 컴포넌트로 작성하는 방법

const 컴포넌트이름 = ({ 사용할 값 }) => { ... } 으로 사용하면 되요!

만약 이런식으로 컴포넌트를 불러오면, 컴포넌트를 안 불러와도 되요!


이 문법은 비구조하 할당이라는 문법이에요!

비구조화 할당의 개념은 비구조화 할당 기본기를 참고하세요!


다만, 이런식으로 작성하게 된다면 State, LifeCycle API를 사용할 수 없어요!

또한, 함수형 컴포넌트는 속도가 빠르기때문에! 단순한 프로그램을 만들때는 함수형 컴포넌트로 만들면 되요!

State

State는 컴포넌트 자기 자신이 들고 있고, 변화가 필요하다면 컴포넌트의 내장 함수 setState 로 값을 변경해줘요

                    render() {
                            return (
                                <div>
                                    <h1>카운터</h1>
                                    <div>값: 0</div>
                                    <button>+</button>
                                </div>
                        );
                    }
이제 setState로 + 버튼을 눌르면 값이 변하는 코드를 작성해봅시다! 
                    state = {
                        numbner: 0
                    }

                    handelInc = () => { {/*화살표 함수를 꼭은 아니지만 ... 사용하면 constructor를 사용하지 않아도 되요!!*/}
                        this.setState({ {/*업데이트 할때는 이 함수를 꼭 사용해야 되요!*/}
                        number: this.state.number + 1
                    })
                    }
                    render() {
                            return (
                                <div>
                                    <h1>카운터</h1>
                                    <div>값: {this.state.number}< {/*State에 정의 된 객체 number의 값을 참조해요!*/}/div>
                                    <button onClick={this.handleInc}>+</button> {/*state의 number 객체의 값이 + 1 씩 증가해요!*/};
                                </div>
                        );
                    }

즉, State는 내부에서 변경 할 때 setState 로 사용해요!

LifeCycle(생명주기) API :D

컴포넌트가 나타날 때, 업데이트 될 때, 사라질 때 중간 중간에 작업을 하고 싶다면 LifeCycle API를 사용해요!

그러면, LifeCycle의 단계를 알아볼게요!

Mounting

Mounting 은 컴포넌트가 브라우저에 나타나는 것을 의미해요!

Mounting의 constructor

constructor 는 생성자 함수로, 처음 브라우저에 나타나게 될때 가장 먼저 실행되는 함수에요

컴포넌트의 State 초기 설정이나, 컴포넌트가 미리작업해야 할것들을 이때 해요

Mounting의 getDerivedStateFromProps

Props의 값을 State에 동기화 하고 싶을때 사용하면 되요!

Props가 바뀔때 사용되요!

Mounting의 componentDidMount

render 함수에서 DOM을 정의해주고, render로 정의한것이 브라우저에 나타나게 되면, componentDidMount가 호출되요

componentDidMount는 외부 라이브러리를 DOM에 그릴때 사용되요!


컴포넌트가 나타나게 된 시점에 어떤 작업을 하겠다라는 걸 명시해 주는 거에요! (e.g API 요청)

Updating

Updating 은 컴포넌트의 Props, State가 바꿨을데 업데이트라고 해요!

Updating의 shouldComponentUpdate

컴포넌트가 업데이트되는 성능을 최적하 하고 싶을때 사용해요 :)

즉, Virtual DOM에 그리는 걸 최적화 할때 사용하죠!


여기서는 true의 값, false을 반환할 수 있는데 이 과정을 컨트롤 할때 사용하는 거죠 👍

Updating의 getSnapshotBeforeUpdate

렌더링하고 난다음의 렌더링 결과물이 브라우저에 출력되기 직전에 실행되는 함수에요!

스크롤의 위치, DOM의 크기, ... 작업을 할때 사용해요!

Updating의 componentDidUpdate

컴포넌트가 업데이트 되었을때 실행하는 함수에요

e.g: State가 바뀌면 그것을 체크하여 컨트롤할때 사용하죠!

Unmounting

Unmounting은 컴포넌트가 화면에서 사라질때를 의미해요!

componentDidMount에서 했던 작업을 없에줄때 사용하면 되요!

Key

Key 는 배열을 렌더링 할때 사용해요!
